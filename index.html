Bestaand project starten

Vanuit de site: klik 🚀 Start bij het project. De browser opent automatisch de claude.ai sessie (kan 10–60 seconden duren).

Vanuit Obsidian: open de project note en klik de 🚀 Start sessie link. Zelfde flow.

⚠️ Skip permissions: gebruik alleen als Claude om bevestiging vraagt bij elke actie en je dat wil overslaan. Voegt --dangerously-skip-permissions toe. Je kunt dit ook permanent instellen op de server: ~/.claude/settings.json → "defaultMode": "bypassPermissions".

Nieuw project aanmaken

Klik "+ Project" rechtsbovenEen panel schuift open vanuit de rechterkant
Vul de naam inLabel is de leesbare naam, Name is de slug (alleen kleine letters en koppeltekens) — wordt gebruikt als tmux sessie naam en GitHub repo
Kies de folder op de serverBijv. /root/projects/mijn-project
Sla op → project verschijnt in de lijstWordt opgeslagen in localStorage — blijft bewaard na reload
Klik 🔨 Maak aan om de GitHub repo + sessie te startenn8n maakt een private repo aan, kloont naar de server en start direct een sessie

Taken toevoegen via de site

Klik "+ Taak" bij een projectEen invoerveld klapt open onder de project card
Typ de taakbeschrijvingOmschrijf wat er gebouwd moet worden — één taak per keer
Druk Enter of klik Toevoegenn8n schrijft - [ ] taak direct naar ## Backlog in de Obsidian project note. Escape sluit het veld.

Daarna: klik ▶️ Analyseer voor het plan, dan ✅ Bouw om de sessie te starten.

Taken plannen en bouwen via Obsidian

Open de project note in ObsidianSla op in 11. Projects/02. Active/{project-naam}.md
Voeg taken toe in ## BacklogElk item als - [ ] wat er gebouwd moet worden
Klik ▶️ Analyseer takenOpent in nieuw tabblad (~30s). Claude analyseert en schrijft plan naar ## Plan.
Bekijk het plan in ObsidianPas eventueel aan. Het plan bevat concrete stappen per taak.
Klik ✅ Bevestig en bouwPlan wordt als TASK.md naar de server geschreven, Claude Code sessie start, browser redirect naar claude.ai

Let op: de SSH credentials voor de Obsidian workflows moeten ingesteld zijn in n8n. Zie Workflow importeren hieronder.

Sessie stoppen of resetten

🛑 Stop kill de tmux sessie en verwijdert de temp file. Hierna kun je opnieuw starten.

↺ Herstart voert Stop en Start na elkaar uit — handig als de sessie vastloopt.

🔄 Reset is de harde reset: kill tmux, verwijdert temp file én kill alle remote-control processen. Gebruik als Stop + Herstart niet werkt.

Workflow importeren in n8n

Nodig bij een herinstallatie of migratie van n8n:

1. n8n → Workflows → Import from file 2. Upload JSON uit /workflows/ in de GitHub repo 3. Controleer credentials: "SSH Claude Code" en "SSH Obsidian/OpenCode/Fabric" moeten bestaan 4. Toggle elke workflow uit → aan om de webhook te registreren

Sessie start maar ik word niet geredirect

Symptoom: de pagina laadt lang en geeft dan "Fout: geen Claude URL gevonden."

Oorzaak: claude remote-control heeft de URL nog niet naar het temp bestand geschreven binnen 60 seconden.

Oplossing: wacht 10 seconden en klik opnieuw op Start. De workflow checkt eerst of er al een URL in de temp file staat — als Claude al draait, redirect je direct.

"Failed to send response. You can try again."

Oorzaak: de verbinding tussen claude.ai en het remote-control proces op de server is verbroken. Dit gebeurt na een timeout, serverherstart of netwerkonderbreking.

Oplossing: klik ↺ Herstart op de projectkaart. Dit stop de sessie en start direct opnieuw.

claude.ai tab geeft een foutmelding na openen

Oorzaak: de sessie URL is verlopen. claude.ai remote-control URLs zijn eenmalig en tijdelijk.

Oplossing: klik ↺ Herstart — er wordt een nieuwe URL gegenereerd.

Stop / Reset knop opent een lege tab

Normaal gedrag — de Stop en Reset knoppen openen de n8n webhook in een nieuw tabblad. n8n geeft een bevestigingspagina terug. Sluit de tab na bevestiging.

Sessie niet gestopt? Gebruik 🔄 Reset: die kill zowel tmux als alle remote-control processen.

Nieuw project mislukt (GitHub repo niet aangemaakt)

Oorzaak: de GitHub Personal Access Token (PAT) in de New Project Session workflow is verlopen of ingetrokken.

1. Ga naar github.com → Settings → Developer settings → Personal access tokens 2. Maak een nieuwe token aan (scope: repo) 3. Open n8n → New Project Session workflow 4. Update de Authorization header en de clone URL met de nieuwe token 5. Sla op en activeer opnieuw

tmux sessie bestaat al maar Claude reageert niet

Oorzaak: de tmux sessie leeft nog maar claude remote-control is gecrasht of gestopt.

Oplossing: klik ↺ Herstart op de projectkaart.

Server niet bereikbaar / n8n geeft timeout

Check 1: is de Proxmox server aan? Host: 192.168.1.34

Check 2: draait n8n nog? Ga naar n8n.jellespek.nl

Check 3: SSH handmatig: ssh root@192.168.1.34

Check 4: docker ps op de server

claude commando niet gevonden op de server

Symptoom: sessie start, temp file blijft leeg, geen URL na 60 seconden.

ssh root@192.168.1.34 which claude # moet een pad teruggeven claude --version # moet werken # Niet gevonden? npm install -g @anthropic-ai/claude-code

Vercel deployment werkt niet na push

Check: ga naar vercel.com → project project-manager → Deployments. Meest voorkomende oorzaak: syntax fout in index.html.
