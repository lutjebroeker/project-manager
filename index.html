Workflows

Start Project Session actief
/webhook/start-project?session=X&folder=Y&mode=normal|dangerous
Herstart of start een sessie. mode=dangerous voegt --dangerously-skip-permissions toe.
Stop Project Session actief
/webhook/stop-project?session=X
Kill tmux sessie en verwijdert de temp file.
New Project Session actief
/webhook/new-project?name=X
Maakt GitHub repo aan (private), clonet naar de server, start sessie.

Bestaand project starten

Vanuit de site: klik 🚀 Start bij het project. De browser opent automatisch de claude.ai sessie (kan 10–60 seconden duren).

Vanuit Obsidian: open de project note en klik de 🚀 Start sessie link. Zelfde flow.

⚠️ Skip permissions: gebruik alleen als Claude om bevestiging vraagt bij elke actie en je dat wil overslaan. Voegt --dangerously-skip-permissions toe.

Typ een project naam Gebruik alleen kleine letters en koppeltekens, bijv. mijn-project

Project toevoegen aan de site

Open index.html lokaal en voeg een object toe aan de projects array (rond regel 270):

{ name: 'project-naam', label: 'Leesbare naam', folder: '/root/projects/project-naam', github: 'https://github.com/lutjebroeker/project-naam', vercel: 'https://project.vercel.app', // weglaten als niet van toepassing }

Commit en push → Vercel deployt automatisch.

Sessie stoppen

Klik 🛑 Stop bij het project op de site, of de Stop sessie link in Obsidian. Dit kill de tmux sessie en verwijdert de temp file op de server. De claude.ai tab gaat vanzelf dood.

Na stoppen kun je direct opnieuw starten — er wordt dan een nieuwe sessie aangemaakt.

Workflow importeren in n8n

Nodig bij een herinstallatie of migratie van n8n:

1. n8n → Workflows → Import from file 2. Upload JSON uit /workflows/ in de GitHub repo 3. Controleer credentials: "SSH Claude Code" moet bestaan 4. New Project Session: vervang {{GITHUB_PAT}} door je GitHub token 5. Toggle workflow uit → aan om webhook te registreren

Sessie start maar ik word niet geredirect

Symptoom: de pagina laadt lang en geeft dan "Fout: geen Claude URL gevonden."

Oorzaak: claude remote-control heeft de URL nog niet naar het temp bestand geschreven binnen 60 seconden.

Oplossing: wacht 10 seconden en klik opnieuw op Start. De workflow checkt eerst of er al een URL in de temp file staat — als Claude al draait, redirect je direct.

"Failed to send response. You can try again."

Oorzaak: de verbinding tussen claude.ai en het remote-control proces op de server is verbroken. Dit gebeurt na een tijdout, serverherstart of netwerk-onderbreking.

Oplossing: stop de sessie via de site en start opnieuw. Er wordt een nieuwe remote-control verbinding en een nieuwe claude.ai URL aangemaakt.

claude.ai tab geeft een foutmelding na openen

Oorzaak: de sessie URL is verlopen. claude.ai remote-control URLs zijn eenmalig en tijdelijk.

Oplossing: stop de sessie en start opnieuw. Er wordt dan een nieuwe URL gegenereerd.

Stop knop lijkt niet te werken

Oorzaak: de fetch() naar n8n kan falen door CORS, maar de sessie is op de server wél gestopt.

Check: probeer daarna opnieuw Start — als er geen bestaande URL gevonden wordt en een nieuwe sessie opstart, was de stop succesvol.

Alternatief: open de Stop-link direct: https://n8n.jellespek.nl/webhook/stop-project?session=project-naam

Nieuw project mislukt (GitHub repo niet aangemaakt)

Oorzaak: de GitHub Personal Access Token (PAT) in de New Project Session workflow is verlopen of ingetrokken.

Oplossing:

1. Ga naar github.com → Settings → Developer settings → Personal access tokens 2. Maak een nieuwe token aan (scope: repo) 3. Open n8n → New Project Session workflow 4. Update de Authorization header en de clone URL met de nieuwe token 5. Sla op en activeer opnieuw

tmux sessie bestaat al maar Claude reageert niet

Oorzaak: de tmux sessie leeft nog maar claude remote-control is gecrasht of gestopt.

Oplossing: stop de sessie via de site en start opnieuw. Start verwijdert de temp file, stuurt C-c naar het claude window en herstart remote-control.

Server niet bereikbaar / n8n geeft timeout

Check 1: is de Proxmox server aan? Host: 192.168.1.34

Check 2: draait n8n nog? Ga naar n8n.jellespek.nl — laadt de UI?

Check 3: SSH handmatig proberen: ssh root@192.168.1.34

Check 4: controleer of Docker nog draait op de server: docker ps

claude commando niet gevonden op de server

Symptoom: sessie start, temp file blijft leeg, geen URL na 60 seconden.

Oorzaak: claude is niet geïnstalleerd of niet in $PATH voor de tmux sessie.

Check via SSH:

ssh root@192.168.1.34 which claude # moet een pad teruggeven claude --version # moet werken

Indien niet gevonden: npm install -g @anthropic-ai/claude-code

Vercel deployment werkt niet na push

Check: ga naar vercel.com → project project-manager → Deployments. Staat er een failed build?

Meest voorkomende oorzaak: syntax fout in index.html. Open de build logs voor details.
